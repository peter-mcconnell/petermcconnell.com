Your first time running perf (part 2) …

2022-12-26 ::
#linux  #perf  #x86  #assembly 

In part 1 we seen the resulting machine instructions from our ‘curl https://www.google.com’ command, but what did it tell us?

Let’s start with setting expectations - here we’re not doing perf analysis on something we think is broken or that needs improved. Curl spending 20% of it’s time on crypto for an HTTPS endpoint isn’t unexpected. The purpose of this article is just to guide you through what it is exactly that you’re looking at and how it may be useful.

Looking over this line by line:

test    %rdi,%rdi

This instruction tests the value in the %rdi register and sets the zero flag (ZF) in the processor’s flags register (%eflags) depending on the result. If the value in %rdi is zero, ZF will be set to 1. If the value in %rdi is non-zero, ZF will be set to 0.

je      1c20ae <OSSL_SELF_TEST_free@@OPENSSL_3.0.0+0x8e>

This instruction is a conditional jump that will only be taken if the zero flag (ZF) in %eflags is set to 1. If ZF is 1, the instruction will jump to the address specified (in this case, 1c20ae). If ZF is 0, the instruction will be skipped and execution will continue with the next instruction in sequence.

and     $0xfffffffffffff000,%r14

This instruction performs a bitwise AND operation between the immediate value 0xfffffffffffff000 and the value in the %r14 register, and stores the result in %r14. A bitwise AND operation sets each bit in the result to 1 if the corresponding bits in both operands are 1, and 0 otherwise.

or      %rax,%r14

This instruction performs a bitwise OR operation between the value in the %rax register and the value in the %r14 register, and stores the result in %r14. A bitwise OR operation sets each bit in the result to 1 if either of the corresponding bits in the operands is 1, and 0 otherwise.

mov     (%r12),%eax

This instruction moves the value at the memory location pointed to by %r12 into the %eax register.

sub     $0x1,%eax

This instruction subtracts the immediate value 0x1 from the value in the %eax register and stores the result in %eax.

cmp     %r13d,%eax

This instruction compares the values in the %r13d and %eax registers and sets the processor’s flags register (%eflags) depending on the result. The cmp instruction does not produce a result, but it updates the flags in %eflags that can be tested by subsequent conditional jump instructions.

jle     1c20b8 <OSSL_SELF_TEST_free@@OPENSSL_3.0.0+0x98>

This instruction is a conditional jump that will only be taken if the sign flag (SF) and the zero flag (ZF) in %eflags are both set to 1, or if the sign flag (SF) is set to 1 and the overflow flag (OF) is set to 0. If these conditions are met, the instruction will jump to the address specified (in this case, 1c20b8). If the conditions are not met, the instruction will be skipped and execution will continue with the next instruction in sequence.

The jle instruction stands for “jump if less than or equal”. It tests the values in the %eflags register to determine whether the value in the first operand (the source operand) is less than or equal to the value in the second operand (the destination operand). If the condition is true, the instruction will jump to the specified address. If the condition is false, the instruction will be skipped and execution will continue with the next instruction in sequence.

The jle instruction is typically used in conjunction with a preceding cmp instruction, which compares the values in the source and destination operands and updates the flags in %eflags accordingly. The jle instruction can then test the values of the flags to determine whether to take the jump or not.

This instruction moves the immediate value 0x1 into the %ecx register.
